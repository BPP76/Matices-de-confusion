2. Objetivo del trabajo
El objetivo de esta práctica es realizar un análisis lo más completo posible del dataset
propuesto, poniendo en práctica de forma integrada todo lo aprendido en la asignatura y, en
general, en el máster proponiendo distintos mini casos.
Se valorará especialmente que combinéis distintas perspectivas y técnicas, por ejemplo:
● análisis exploratorio de datos,
● visualización,
● grafos,
● series temporales,
● NLP,
● clasificación,
● clustering,
● feature engineering,
● interpretación de resultados,
● aplicación práctica de los hallazgos.
● …
Podéis utilizar Deep Learning y cualquier otra herramienta o software que consideréis
adecuado, como por ejemplo R, Python, Tableau, Gephi, etc. En todos los casos, deberéis
adjuntar los ficheros correspondientes a las herramientas utilizadas.

7. Requisitos técnicos
Para que la entrega sea válida, debéis cumplir los siguientes requisitos:
● El notebook debe poder ejecutarse de principio a fin sin intervención manual.
● Todas las rutas utilizadas en el código deben ser rutas relativas.
● Se recomienda trabajar con una estructura limpia en la que el notebook actúe
como punto de entrada y lance procesos apoyándose en archivos .py bien
organizados.
● Podéis tener scripts auxiliares o propios fuera del notebook, pero también deben
incluirse en la entrega.
● Es obligatorio usar Git durante el desarrollo del proyecto.
● Se valorará el uso correcto y ordenado del repositorio.
Además, el notebook debe estar organizado por secciones, bien estructurado y
documentado.
8. Requisitos del informe
El informe tendrá una extensión máxima de 20 páginas.
No cuentan dentro de ese límite:
● la portada,
● el índice,
● las referencias bibliográficas,
● los anexos.
Figuras, tablas y anexos
Todas las figuras del informe (gráficos, tablas, dibujos, esquemas, etc.) deben:
● estar mencionadas explícitamente en el texto mediante una referencia, por
ejemplo: Figura 8, Tabla 3, etc.;
● aparecer en el anexo.
El límite de 20 páginas se aplica al cuerpo principal del texto.

Un análisis inicial del conjunto de datos que permita entender su estructura, calidad,

El trabajo debe incluir un mínimo de 4 casos de uso o iniciativas.
Cada iniciativa debe tener entidad propia y presentar, al menos, los siguientes elementos:
● planteamiento inicial,
● desarrollo,
● exploración de variables/features,
● feature engineering,
● métricas de evaluación o error, cuando proceda,
● estudio de reglas, clusters o patrones, si aplica,
● interpretación de resultados,
● análisis de errores,
● aplicación práctica,
● conclusiones,
● mejoras y próximos pasos.
